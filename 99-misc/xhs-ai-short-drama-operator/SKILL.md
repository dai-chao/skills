**步骤2：AI添加字幕（保留原音频）**

**极其重要：即梦生成的原视频通常带有BGM，处理时必须保留原音频轨道！**

**音频处理原则：**
1. 不要在没有确认的情况下使用 `-an`（去掉音频）
2. 如果要添加AI口播，需要先把原视频BGM音量降低，再把语音混进去
3. 混音公式：`原BGM * 0.3 + 口播语音 * 1.5`

**当ffmpeg编译时缺少drawtext滤镜**（常见情况），使用Python PIL生成字幕帧 + ffmpeg overlay的方案：

**A. 生成字幕帧（Python）：**
```python
from PIL import Image, ImageDraw, ImageFont
import os

os.makedirs('subs', exist_ok=True)
font = ImageFont.truetype("/System/Library/Fonts/PingFang.ttc", 40)

# 按时间轴定义字幕 (帧号_start, 帧号_end, 文字)
subtitles = [
    (0, 90, "第一句文案"),
    (90, 180, "第二句文案"),
    # ...
]

for frame_idx in range(total_frames):
    img = Image.new('RGBA', (834, 1112), (0, 0, 0, 0))
    draw = ImageDraw.Draw(img)
    for start, end, text in subtitles:
        if start <= frame_idx < end:
            bbox = draw.textbbox((0, 0), text, font=font)
            tw, th = bbox[2] - bbox[0], bbox[3] - bbox[1]
            x, y = (834 - tw) // 2, 1112 - th - 100
            draw.rectangle([x-20, y-10, x+tw+20, y+th+10], fill=(0, 0, 0, 180))
            draw.text((x, y), text, fill=(255, 255, 255), font=font)
    img.save(f'subs/{frame_idx:04d}.png')
```

**B. 字幕帧转透明视频：**
```bash
ffmpeg -framerate 60 -i subs/%04d.png \\
  -c:v qtrle -pix_fmt argb -t 15 subtitle_track.mov
```

**C. 叠加字幕到原视频（保留原音频）：**
```bash
# 基础叠加：保留原音频
ffmpeg -i original.mp4 -i subtitle_track.mov \\
  -filter_complex "[0:v][1:v]overlay=0:0[v]" \\
  -map "[v]" -map 0:a \\
  -c:v libx264 -crf 23 -preset fast -pix_fmt yuv420p \\
  -c:a copy final_video.mp4
```

**D. 如果还要添加AI口播语音（混音版）：**
```bash
ffmpeg -i original.mp4 -i subtitle_track.mov -i voiceover.wav \\
  -filter_complex "
    [0:v][1:v]overlay=0:0[v];
    [0:a]volume=0.3[bgm];
    [2:a]volume=1.5[voice];
    [bgm][voice]amix=inputs=2:duration=first:dropout_transition=2[aout]
  " \\
  -map "[v]" -map "[aout]" \\
  -c:v libx264 -crf 23 -preset fast -pix_fmt yuv420p \\
  -c:a aac -b:a 192k final_video.mp4
```

**E. 清理临时文件：**
```bash
rm -rf subs subtitle_track.mov voiceover.wav voiceover.ogg
```

## macOS 本地 Safari 半自动化操作（实测经验）

当远程浏览器因 IP 风控（30012）无法访问 creator.xiaohongshu.com 时，可在用户本地 Mac 的 Safari 上进行**有限自动化**：

### 前提条件
1. Safari → Settings → Advanced → "Show Develop menu in menu bar" 开启
2. Safari → Settings → Developer → **"Allow JavaScript from Apple Events"** 必须勾选
3. 用户需手动登录小红书创作者后台

### 可用操作
- `open -a Safari 'https://creator.xiaohongshu.com/new/home'` — 打开首页
- `osascript` + `tell application "Safari" to tell current tab of window 1 to do JavaScript "..."` — 执行 JS

### 关键限制（实测发现）
1. **IP 风控差异**：远程浏览器环境几乎必被 300012 拦截；用户本地 Safari 使用正常网络，可正常访问
2. **文件上传不可自动化**：系统文件选择对话框无法绕过，图片/视频必须由用户手动拖拽或选择
3. **编辑器无法文本注入**：创作者后台的富文本编辑器使用 iframe + 自定义渲染（Canvas/React 封装），通过 `contentDocument` 或 `querySelector('input/textarea')` 无法定位。`document.body.innerHTML/innerText` 在 iframe 内返回空。因此**标题、正文、标签必须通过用户手动复制粘贴填入**
4. **发布流程差异**：
   - 图文模式（target=image）：必须先上传图片才会出现标题/正文编辑区
   - 长文模式（target=article）：有独立入口 `https://creator.xiaohongshu.com/publish/publish?target=article`，无需图片，可直接粘贴文字

### 可自动化的极限
- ✅ 打开指定页面（首页、发布页）
- ✅ 点击简单的导航按钮（如"发布图文笔记"）
- ✅ 探测页面 URL 和标题变化
- ✅ 用 Finder `open -R` 高亮图片文件，方便用户拖拽
- ❌ 上传图片/视频
- ❌ 注入标题、正文、标签文本
- ❌ 点击"发布"按钮（未验证，但大概率受前端封装影响）

### 推荐工作流
1. AI 准备好文案 + 图片素材，保存到本地路径（如 `/Users/chao/Desktop/xiaohongshu`）
2. 用 `open -a Safari` 打开小红书发布页
3. 用 `open -R` 在 Finder 中高亮素材，方便用户拖拽上传
4. 将文案整理为"复制即用"格式，由用户手动粘贴到对应区域

## 常见坑与误解（实战经验）

### 1. 浏览器自动化限制
- **误解**：用户认为AI可以直接操作他本地浏览器
- **事实**：自动化工具运行在隔离环境中，无法访问用户本地浏览器的session/cookie
- **建议**：提前说明工具限制，让用户在APP/本地浏览器中手动完成登录

### 2. 短信验证码问题
- **误解**：用户提供手机号后期待AI能收到验证码
- **事实**：AI环境无法接收手机短信，发送验证码按钮在风控环境下常常失败
- **建议**：让用户自己收验证码并告知AI，或直接让用户登录好后再给AI操作

### 3. 图像质量误区
- **误解**：用户说"你自己能生成也行"时，可能误以为AI有专业绘图能力
- **事实**：代码绘图（PIL等）仅为简单几何图形，远不如专业AI绘图工具
- **建议**：第一时间询问用户是否有AI绘图平台账号，如即梦/Midjourney，如果没有则提供提示词让用户手动生成

### 4. 即梦登录流程
- **误解**：以为即梦支持手机号验证码登录
- **事实**：即梦强制抖音OAuth扫码授权，必须用户主动用抖音APP扫码
- **建议**：不要尝试用手机号登录即梦，直接让用户扫码授权

### 5. 视频制作工作流的工具检查

**必须检查的工具**：
```bash
which ffmpeg
/usr/bin/python3 -c "from PIL import Image; print('Pillow OK')"
```

**ffmpeg必需支持的编码器**：
- `libx264`: H.264视频编码
- `qtrle`: 快速无损视频编码（用于透明字幕轨）

**ffmpeg缺少drawtext滤镜的判断**：
如果运行：
```bash
ffmpeg -f lavfi -i color=c=black:s=100x100 -vf "drawtext=text='test':fontsize=30" -frames:v 1 test.png
```
报错 `No such filter: 'drawtext'`，则必须使用Python PIL生成字幕帧的方案。

### 6. 图片质量评估

**在用户评价之前自检**：
- Python PIL生成的头像/背景/视频封面 → 质量不达标
- 必须告知用户"这是草图，需要即梦重生"
- 不要在用户表示不满意时还指望代码绘图

**质量达标标准**：
- 文字清晰可读
- 人物形象具有专业AI绘图的细节和质感
- 不是几何简笔画
- 有明确的网感/电影质感
