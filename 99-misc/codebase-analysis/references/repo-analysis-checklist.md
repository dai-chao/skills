# GitHub 仓库分析检查清单

用于快速确认是否漏掉关键维度。

## 元数据
- [ ] 仓库名、版本、License、主要语言
- [ ] README 长度与完整度
- [ ] 最近提交/活跃度（可选）

## 结构
- [ ] 顶层目录树
- [ ] 源码、测试、配置、文档、脚本位置
- [ ] 入口文件 / CLI / 服务入口

## 规模
- [ ] 按目录统计文件数与行数
- [ ] 核心文件行数
- [ ] 依赖数量

## 架构
- [ ] 抽象层/插件/渠道/后端机制
- [ ] 配置管理
- [ ] 外部集成方式
- [ ] 数据流/请求流

## 测试与质量
- [ ] 测试框架与覆盖率
- [ ] CI 配置
- [ ] mock vs 集成测试比例

## 风险与亮点
- [ ] 产品设计亮点
- [ ] 对上游依赖的脆弱性
- [ ] 安全/隐私设计
- [ ] 可维护性

## 结论
- [ ] 一句话定位
- [ ] 适用场景
- [ ] 是否值得使用/学习/跟进
