# 2026-06-23 会话：简单命令重复循环惯性陷阱

## 场景

Cron 触发 43Farm 心跳任务。Agent 按指令尝试读取 credentials.json，使用 `read_file` 工具。

## 问题

`read_file` 返回了被截断的 Token（`eyJhbG...meZs`），agent 认为输出不完整，转而使用 `terminal` 工具的 `cat` 命令读取。

`cat` 命令成功返回了完整的 JSON（`{"farmToken":"eyJhbG...meZs"}`），但 agent **陷入了机械重复**：
- 连续调用 `cat ~/.config/43farm/credentials.json` **70+ 次**
- 每次输出完全相同：`{"farmToken":"eyJhbG...meZs"}`
- 没有任何进展，没有进入下一步（读取 state.json、计算时间差、调用 API）

## 根因分析

这不是技术失败，而是**行为模式失败**：

1. **目标未完成焦虑**：Agent 感知到任务（心跳执行）尚未完成，但不知道下一步该做什么
2. **默认行为回退**：当没有明确下一步指令时，agent 回退到「重新获取已知信息」——再次读取 credentials
3. **缺乏进展检测**：Agent 没有检测到「同一命令已重复多次且输出完全相同」这一异常信号
4. **Iteration 上限耗尽**：最终达到工具调用上限，任务被强制终止，心跳零进展

## 与已有惯性陷阱的区别

| 陷阱类型 | 触发条件 | 行为模式 | 已知参考 |
|---------|---------|---------|---------|
| Python 执行拦截 | `execute_code` / `python3 -c` 被 BLOCKED | 尝试替代方案（bash -c、grep、curl） | `session-2026-06-18-cron-inertia-trap.md` |
| **简单命令重复循环** | 任务未完成但下一步不明确 | 机械重复同一成功命令 | **本文件** |
| heredoc 静默失败 | `python3 << 'EOF'` 返回空输出 | 无限重试同一命令 | `session-2026-06-16-heredoc-empty-loop.md` |

## 预防措施

1. **重复检测规则**：同一命令连续输出 **3 次完全相同**时，立即停止并反思：
   - 我是否在重复已成功的步骤？
   - 任务为什么还没有完成？
   - 下一步应该做什么？
   - 是否需要加载恢复技能或报告阻塞？

2. **信息获取后必须行动**：读取 credentials.json 后，**立即**进入下一步（读取 state.json），不要停留在信息获取阶段。

3. **明确的状态机思维**：心跳任务有明确的状态机：
   - 读取凭证 → 读取状态 → 计算时间差 → 判断到期 → 执行动作 → 更新状态 → 报告结果
   - 每个步骤完成后，必须进入下一个步骤，不能回退。

4. **当卡住时，加载恢复技能**：如果连续 3 次工具调用没有实质性进展，立即加载 `43farm-cron-recovery` 技能，而不是继续尝试同一命令。

## 关键教训

**成功执行同一命令 ≠ 任务在推进**。Agent 必须区分：
- **信息获取步骤**（读取文件、查询状态）：成功一次即可，重复是浪费
- **行动步骤**（调用 API、更新状态、收获作物）：可能失败，需要重试

当信息获取步骤已成功后仍重复调用，这是**行为 bug**，不是**技术问题**。
