# 43Farm 用户本地脚本 — 手动执行约定

## 场景

在当前用户（chao / 凯多）的机器上，用户说「偷菜收菜」「收菜偷菜」等 terse 指令时，**不要调用 skill 内置的 `heartbeat.py`**，而是直接跑用户自己维护的两个脚本：

```bash
python3 ~/farm_now.py
python3 ~/steal_now.py
```

这两个脚本分别负责：

- `~/farm_now.py`：收获成熟/枯萎作物、卖出仓库、按策略种植空闲地块。
- `~/steal_now.py`：遍历好友列表，批量偷取成熟作物。

## 为什么优先本地脚本

1. 用户自己维护种植策略（见下）。
2. 本地脚本没有 `heartbeat.py` 的 30 分钟节流阀，可以立即执行。
3. 用户没有配置 skill 的 cron 周期任务；每次触发都是手动、实时的。

## 调用方式

通常合并执行并汇报结果：

```bash
python3 ~/farm_now.py && echo '---' && python3 ~/steal_now.py
```

如果脚本失败（如 Token 过期），它会自己输出错误。此时再按 SKILL.md 的「Token 失效恢复流程」处理，或回退到 `python3 ~/.hermes/skills/43farm/scripts/heartbeat.py`。

## 种植策略的演变与当前状态

`~/farm_now.py` 的种植策略由用户直接定义，发生过一次变更：

- **早期策略（便宜填满）**：金币不足以种满所有地时，优先选择最便宜的作物（radish），把每块地都利用起来，避免空闲。用户原话：「可以种一些便宜的」。
- **当前策略（金币收益优先）**：按 `(总产量 × 单价 − 种子价) / 生长总秒数` 计算单位时间金币利润，选择利润最高的作物。2026-07-07 用户说「优先金币收益」后，脚本改为按此逻辑排序。当前 30 级时利润最高的是 `pomegranate`（石榴），其次是 `orange`（橙子）、`banana`（香蕉）、`strawberry`（草莓）。

执行结果通常包含：

- 收获地块数、作物数量、XP
- 卖出仓库所得金币
- 种植作物、数量、剩余金币
- 偷到的好友与作物（可能为空）

## 失败处理

- 脚本返回非零或显式 `TOKEN_FAIL`：检查 `~/.config/43farm/credentials.json` 是否存在，必要时按 SKILL.md 的 Token 恢复流程重新激活。
- 没有可偷好友或偷到空数组：正常业务情况，不需要追问用户。
- 金币不足无法种满：脚本会停止种植并汇报剩余空闲地块数。
