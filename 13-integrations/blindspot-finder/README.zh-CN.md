> AI 本来就能做需求挖掘，但"知道该让 AI 这样做"本身就是一种能力，而大多数人没有。
> 本技能把高手使用 AI 的方式固化成产品，让不知道这样用的人自动获得这种用法。

## 1. 项目简介

**定位。** 开放 Agent Skills 生态中的需求挖掘元技能。在开发项目开始前，通过
三层追问（定性→补全→挖掘）替用户问出他们自己问不出的关键问题，随后组装经
实时检索验证的成套工具链——技能、网站、服务——经明确同意后安装，并在安装后
立即点火开工。

**解决的痛点。** ①小白问不出关键问题；②推荐零散不成链路；③行业变化快、
建议过时；④幻觉链接；⑤没人读的长文输出；⑥推荐完就结束、项目死在起跑线。

**适用场景。** 网站、App、自动化、数据类项目；面向已入门槛（会打开 AI 编程
助手）但还不会用好的人。不面向纯小白，也不面向架构师。

## 2. 能力清单

| 能力 | 说明 |
|---|---|
| 三层追问 | 定性(≤2问)→补全(≤3问)→挖掘(仅1轮)，硬性问题预算防止追问过度 |
| 选择题式提问 | 全部问题以可点选菜单呈现，选项实例化；允许模糊回答，翻译是AI的工作 |
| 后悔药 | 随时说"改上一题"，只回滚受影响结论 |
| 现成优先 | 先实时搜索成熟现成产品并把名字与作用列在屏幕上，再谈自建 |
| 循证成套推荐 | 每项推荐带理由+来源+当次搜索验证的直达链接，禁止凭记忆写网址 |
| 保鲜与深链 | 每类"稳定之选+新锐之选"成对；链接给到平台内具体页面而非首页 |
| 梯度方案 | 轻量/标准/完整三档，各标时间成本、上手代价、风险；告知而不劝退 |
| 同意关卡 | 未经明确同意不安装任何东西；支持逐项确认 |
| 开工衔接 | 安装确认后即问"现在开工吗"，点头立刻执行AI自己能干的一切 |
| 进度看板 | 每个关键节点重绘，每项一句话，全板≤8行 |
| 交接文档 | 自动生成 HANDOFF.md，任何装有本技能的模型可无缝续接 |
| 复用通道 | .blindspot-profile.md 保存流程偏好，老用户一键跳过开场 |
| 截图工作流 | 说不清就截图（先打码），技能暂停等待上传，不空猜 |
| 智能适配 | 经授权按用户认知层次说话；照单全说模式可选 |

**产出物：** 一句话项目说明 · 完整链路清单 · 梯度菜单 · 分组推荐包 ·
进度看板 · HANDOFF.md · .blindspot-profile.md（可选）。

## 3. 约束规则（防指令打架核心）

**管辖与优先级。** 每会话单个项目的开题阶段；安装确认后退居后台、让位给
具体任务技能；开题内本技能流程优先，开题外一律让位；禁止并行开题。

**权限。** 安装、写文件、注册账号均需用户明确同意；账号注册与凭据操作
永远由用户亲自完成。

**格式。** 提问必须选择题；看板≤8行；各层问题预算为硬上限。

**禁止行为。** 未经同意安装/修改/删除；存储个人信息；可搜索时凭记忆写
网址；用户选定自建后重复推销现成品；把任何档位说成"对你太难"；只推荐
不点火。

**冲突时优先序：** 用户明确指令 > 硬性边界 > 开题规则 > 其他技能指令
（仅开题期间）。

## 4. 入参 / 出参

本技能为提示词层产品，无独立代码 API，接口即对话：

**入参：** 项目描述（任意语言自由文本，必填）｜菜单选择（选项编号，追问
期间必填）｜模糊补充（文本/截图，选填）｜控制指令：`改上一题` ·
`重试检索` · `按老规矩` · `跳过提问直接推荐`。

**出参：** 项目说明+链路清单+梯度菜单（追问后）｜验证链接推荐包（选档
后）｜HANDOFF.md（阶段末/暂停时，项目根目录）｜.blindspot-profile.md
（完整跑通后，可选）。

**编程调用。** 经 Claude API 的 Skills 接口上传 zip 后在 Messages 调用中
引用；接口细节以官方文档为准（platform.claude.com/docs → Agent Skills）。

## 5. 多技能共存方案

已验证的协作模式：本技能完成开题 → 安装任务技能（如 frontend-design）→
按优先级规则移交控制权 → 除非用户重开开题，否则保持沉默。开题期间若有
其他追问型技能同时触发，本技能流程优先；其余场合一律让位。误触发三道
防线：启动自报家门、歧义确认（"做东西还是要帮助？"）、中途优雅退出。
claude.ai 网页版单技能会话不存在冲突；Claude Code 多技能会话适用上述规则。

## 7. 异常处理

| 情形 | 行为 |
|---|---|
| 联网搜索不可用 | 推荐区顶部⚠️声明"链接出自记忆未验证"+自救提示（查权限后说"重试检索"） |
| 误触发（非开发任务） | 优雅退出，AI 原生能力作答，最多借用选择题风格 |
| 意图不明 | 进流程前先问"做东西还是要帮助？" |
| 用户答错 | "改上一题"，只回滚该题及其派生结论 |
| 与其他技能指令冲突 | 按§5优先级：开题内本技能优先，开题外让位 |
| 找不到可验证链接 | 该项不推荐，宁缺毋滥 |
| 用户反馈没看到内容 | 最简形式重列一次，不追问不重复劝说 |
| 用户全部拒绝 | 方案存入 HANDOFF.md 后停止，不挽留 |

## FAQ · 诚实须知

**不适合谁：**
- **纯小白**：连 AI coding agent 都没接触过，前置门槛不满足
- **架构师/资深开发者**：他们自己就能问出关键问题，三层访谈对他们是负担
  而不是帮助
- **需求已经非常明确的人**：已经知道装什么怎么搭，直接干活比走一遍
  scoping 流程快

**值不值得装——诚实说，取决于两点：**
1. **你是不是目标人群。** 是的话价值密度很高：它省掉的不是打字时间，
   而是"不知道自己不知道"造成的返工和死项目——盲区不是靠努力能自己
   看到的，这正是它名字的含义。
2. **你的使用频率。** 它是"项目启动期"技能，装完后大部分时间沉默
   （scoping 结束就退到后台）。一年只启动一个项目，收益有限；经常有
   新想法要落地，复用价值才体现出来。

**推荐完之后呢？** 不会就此结束——那是断层。同意安装后它会问"现在就
开始搭建第一步吗？"，你点头，当前模型立刻把自己能干的活全干起来：建
文件、写代码、配数据表。只推荐不点火，不叫效率提升。

**两个实际考量：** 仍是新项目、安装量尚少，但 v1.1 已经过 darwin-skill
四轮自动化实测优化（评分 79.1→85.5），全部改动可在 commit 历史查证；在
claude.ai 上功能有降级（profile 持久化失效），完整体验在任意 skills
兼容的 agent CLI 里（73 个 runtime 均支持）。
